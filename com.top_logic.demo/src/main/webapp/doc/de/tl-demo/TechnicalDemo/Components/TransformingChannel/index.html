Sichten in TopLogic ermöglichen die Navigation in einem Datenbestand, indem die Selektion in einer Selektionskomponente mit dem Modell einer Darstellungskomponente verknüpft wird. Jede Komponente hat mindestens einen "Modell"-Kanal, über den sie ihr Kontext-Objekt bezieht. Eine Tabelle beispielsweise verwendet das Kontext-Modell, um daraus die angezeigten Zeilen zu berechnen. Ein Formular dagegen zeigt die Attribute des Kontext-Modells an und ermöglicht deren Bearbeitung.

Eine Tabelle hat neben ihrem Modell-Kanal auch einen Selektionskanal, über den sie die von einem Nutzer selektierte Zeile publiziert. Verbindet man den Selektionskanal einer Tabelle mit dem Modell-Kanal eines Formulars, erhält man eine Sicht, in der das Formular neben der Tabelle immer die Details (alle Attribute) des in der nebenstehenden Tabelle selektierten Zeilenobjektes anzeigt.

Diese Verbindung kann mehrstufig sein. So kann eine Tabelle ihr Kontext-Modell von einem nebenstehenden baum empfangen und die Selektion an ein Formular weitergeben.

In vielen Fällen ist es ausreichend, Komponentenkanäle direkt miteinander zu verbinden. Ist die empfangende Komponente ein Formular, kann es aber beispielsweise sein, dass man nicht direkt das in einer nebenstehenden Tabelle selektierte Objekt anzeigen möchte, sondern die Details einer daraus abgeleiteten Eigenschaft. In diesem Fall muss bei der Weiterleitung der Selektion (beispielsweise in einer Tabelle) zu einem Modell (beispielsweise eines Formulars) eine Transformation stattfinden.

Eine solche Modell-Transformation lässt sich in der Modell-Einstellung beispielsweise einer Formular-Komponente vornehmen. Mit der Einstellung "Transformation" kann die Quell-Komponente und deren Kanal gewählt werden. Zusätzlich lässt sich eine Funktion konfigurieren, welche das so erhaltene selektierte Objekt in das Modell-Objekt der Zielkomponente umwandelt.

Die hier dargestellte Sicht demonstriert eine Transformation in einer Kanal-Verknüpfung, indem nicht die Selektion in der links angezeigten Tabelle direkt in dem nebenstehenden Formular angezeigt wird, sondern das Objekt, dass im Attribut "singelStructure" des selektierten Zeilenobjekts gespeichert ist.

Eine weitere Möglichkeit ist die Kanalkombination um die Werte aus mehreren Komponentenkanälen zu einem Wert zu kombinieren.
