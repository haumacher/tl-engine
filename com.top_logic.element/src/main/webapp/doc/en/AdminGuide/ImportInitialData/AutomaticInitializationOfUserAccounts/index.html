Wird die Anwendung erstmalig gegen eine leere Datenbank gestartet, so wird die Datenbank automatisch initialisiert. Dabei besteht die Möglichkeit, bei dieser Initialisierung eine zuvor beschriebene Menge von Benutzerkonten anzulegen.

In der Regel sind im Auslieferungsumfang initiale Administratoren und/oder Test-Konten enthalten, so dass diese Konfiguration keiner Anpassung bedarf.

Die Menge der initial anzulegenden Benutzerkonten wird in folgenden Konfigurationsdateien definiert:

InitialTestUsers.xml

KBDataInitialTestUsers.xml

Die in diesen beiden Dateien spezifizierten Benutzerkonten werden beim ersten Start der Applikation automatisch angelegt.InitialTestUsers.xml
