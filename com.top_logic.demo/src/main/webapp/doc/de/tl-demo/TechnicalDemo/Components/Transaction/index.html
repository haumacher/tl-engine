Die hier dargestellte Sicht demonstriert eine benutzerdefinierte Transaktion, bei der der Nutzer vorher zu einer Eingabe von zusätzlichen Argumenten aufgefordert wird.

Der hier definierte Knopf "Selektion duplizieren" ist über die Konfigurationsoption "Dialog hinzufügen / Transaktion mit Eingabe" definiert. Die beispielhaft gezeigte Funktionalität dupliziert das in der Tabelle selektierte Zeilenobjekt und gibt ihm dabei einen neuen Namen. Dieser neue Name für neu angelegte Objekt kann durch den Nutzer vorher in einem Dialog eingegeben werden. Das System schlägt für den neuen Namen einen Standardwert vor, der sich aus dem Namen des zu kopierenden Objektes ableitet.
