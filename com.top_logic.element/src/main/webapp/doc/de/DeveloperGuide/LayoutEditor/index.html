Mit dem Layout-Editor lassen sich neue Sichten in einer Anwendung erstellen. Eine Sicht zeigt einen gewissen Ausschnitt des Fachmodells der Anwenung in einer für den jeweiligen Anwendungsfall optimierten Weise an. Die Sichten der Anwendung werden in Reitern oder Kacheln strukturiert. Ein Reiter oder eine Kachel kann dabei wieder

  • Tabellen
  • Bäume
  • Formulare
  • Diagramme
  • Kacheln
  • oder weitere Reiter

in beliebiger Schachtelungstiefe enthalten.

Designmodus

Um neue Sichten in einer Anwendung zu erstellen, muss diese in den Design-Modus geschaltet werden. Nur ein Administrator hat diese Option zur Auswahl. Um die so konfigurierten Sichten für andere Nutzer zugänglich zu machen, muss anschließend das Layout veröffentlicht werden.

Im Design-Modus erhalten Reiter und Sichten neue Einträge in ihren Menüs, mit dem ihre Konfigurationen angepasst oder neue Reiter und Kacheln hinzugefügt werden werden können. Der Rest dieses Kapitels beschäftigt sich mit den Möglichkeiten der Layout-Konfiguration.

Layouts veröffentlichen

Nach Abschluss der Layout-Konfiguration durch einen Administrator-Nutzer müssen die so konfigurierten Sichten veröffentlicht werden, damit andere Nutzer Zugriff darauf erhalten.

Nach dem Veröffentlichen der Layouts kann der Design-Modus wieder verlassen werden, um die Anwendung im Produktivmodus weiter zu nutzen. Sind andere Nutzer während der Layoutkonfiguration eingeloggt, sehen sie die Änderungen am Applikationslayout erst nach dem nächsten Login in die Anwendung.

Layouts exportieren

Wird eine Anwendung aus einer lokalen Entwicklungsumgebung heraus betrieben, so können die in-app konfigurierten Layouts über "Layouts exportieren" als XML-Konfiguration zurück ins Dateisystem der Entwicklungsumgebung geschrieben werden um so in einen regulären Deploymentprozess einfließen zu können. Bei einer Server-Installation ist dieser Punkt nicht verfügbar.
