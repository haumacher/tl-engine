Module (oder Pakete) sind eine Möglichkeit, große Modelle zu strukturieren. Alle in einem Fachmodell definierten Typen leben in einem Modul. TopLogic bringt im Auslieferungszustand bereits eine Reihe von Modulen mit. Alle Ergänzungen, die in einer Anwendung definiert werden, sollten in einem oder mehreren eigenen Modulen definiert werden.

Ein neues Modul wird im Modul-Baum des Modell-Editors angelegt.

Zur Definition eines Moduls gehört mindestens ein technischer Name. Dieser Modul-Name wird Teil des Names aller in diesem Modul definierten Typen. Der technische Name sollte nur aus Kleinbuchstaben und Punkten ohne Leerzeichen bestehen. Zusätzlich kann das Modul einen Anzeigenamen erhalten, der statt des technischen Namens z.B. im Modul-Baum des Modell-Editors angezeigt wird.

Der Modell-Editor zeigt immer ein Klassen-Diagramm für ein Modul an. Dieses Diagramm enthält die in diesem Modul definierten Typen und alle direkten Beziehungen, die diese Typen zu Typen aus anderen Module haben. Direkt nach Anlage eines neuen Modul ist das Diagramm leer. Es kann danach mit neuen Typen gefüllt werden.
