Here you can test the PDF export.

There are different types A, B, C, D (in module test.pdfExport) where A is a superclass of B, C, and D.

  • In A a form definition is specified and a PDF export definition.
    When A is exported, the PDF export definition of A is used.
  • In B a PDF export definition is given.
    If B is exported, the PDF export definition of B is used, but the form of A is displayed.
  • Nothing is configured in C.
    The definition of A is used for both display and export.
  • Nothing is configured in D. A local PDF export definition is set in the component.
    Here the configured PDF export definition is used when exporting, elsewhere the definition of A would be used.

Snapshots

The print command in the snapshot table can be used to print the currently displayed object to a PDF document and store the resulting PDF file in the system. The snapshot table shows all printed versions of the current object. The PDF download of the PDF file can be used to trace what value the object had at the time of snapshot creation.

Side note: This functionality is just a demo for the PDF export feature in TLScript. A traceable history of objects could be much more easily realized using TopLogic's built-in data historization.
