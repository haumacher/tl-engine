Here you can test the PDF export.

There are different types A, B, C, D (in module test.pdfExport) where A is a superclass of B, C, and D.

  • In A a form definition is specified and a PDF export definition.
    When A is exported, the PDF export definition of A is used.
  • In B a PDF export definition is given.
    If B is exported, the PDF export definition of B is used, but the form of A is displayed.
  • In C nothing is configured.
    The definition of A is used for both display and export.
  • Nothing is configured in D. A local PDF export definition is set in the component.
    Here, the configured PDF export definition is used when exporting, in other places the definition from A would be used.
