Syntax

	$input.parseCSV()

Beschreibung

Interpretiert die Eingabe als CSV formattierten Text.

​Parameter

Name Typ Beschreibung Pflicht Default
input Binärwert Text im CSV-Format ja -
parsers Map Formatierungsangaben für die einzelne spalten Nein null
columnSeparator Char Trennzeichen für Zeileneinträge Nein ','
lineSeparator String Trennzeichen für die einzelne Zeilen Nein "\n"
quoteChar Char

Zeichen, das zum Zitieren von Werten verwendet wird, die Anführungszeichen oder Zeilenumbrüche enthalten.

 Nein '"'
trimSpaces Boolean true: wenn Leerzeichen um Trennzeichen automatisch gekürzt werden sollen. Nein false
raw Boolean

raw = true: Die Funktion liefert die Rohdaten als Liste von Listen mit

Header zurück.

sonst: Die Funktion liefert eine Liste von Maps, wobei die Spaltennamen

(aus dem Header) als Key verwendet werden.

 Nein false/null

Rückgabewert

Typ: Liste

Erzeugt pro Zeile in der Eingabe eine Liste/Map und fasst anschließend diese Listen in einer Liste zusammen.

Beispiele

Beispiel 1:

{
    toParse = binary(
        name: "test.csv",
        data: "date; name; number
        15.07.2024; test; 5
        16.07.2024; test2; 2; etc.
        17.07.2024",
        contentType: "text/csv",
        encoding: "utf-8"
    );
    result = $toParse.parseCSV(raw:true, columnSeparator: ";");
    $result[0]
}

Ausgabe: ["date", "name", "number"]

Der Aufruf von parseCSV(raw:true, columnSeparator: ";") liefert die Liste [["date", "name", "number"], [" 15.07.2024", "test", "5"], [" 16.07.2024", "test2", "2", "etc."],[" 17.07.2024"]]. Also mit Leerzeichen vor das Datum. Wenn wir diese Leerzeichen los werden möchten, müssen wirparseCSV(raw:true, columnSeparator: ";", trimSpaces:true) aufrufen.

Beispiel 2:

{
    toParse = binary(
        name: "test.csv",
        data: "date; name; number
        15.07.2024; test; 5
        16.07.2024; test2; 2; etc.
        17.07.2024",
        contentType: "text/csv",
        encoding: "utf-8"
    );
    result = $toParse.parseCSV(columnSeparator: ";");
    $result
}

Ausgabe: [{date= 15.07.2024, number= 5, name= test}, {date= 16.07.2024, number= 2, name= test2}, {date= 17.07.2024, number=, name=}]

Der Aufruf von parseCSV(columnSeparator: ";") liefert eine Liste von Maps. Dabei wird die erste Zeile als Header gewählt und deren Einträge als Schlüssel für die Maps verwendet. Anschließend wird für jede weitere Zeile aus deren Einträgen gemäß den Schlüsseln des Headers der zugehörigen Map erzeugt, wobei überschüssige Einträge der Zeile ignoriert und fehlenden Einträgen der Zeile mit leeren Strings aufgefüllt werden.

Beispiel 3:

{
    toParse = binary(
        name: "test.csv",
        data: "date; name; number
        15.07.2024; test; 5
        16.07.2024; test2; 2; etc.
        17.07.2024",
        contentType: "text/csv",
        encoding: "utf-8"
    );
    result = $toParse.parseCSV(columnSeparator: ";", trimSpaces:true, 
        parsers: {"date": x-> dateFormat("dd.MM.yy").format(dateFormat("dd.MM.yyy").parse($x)), 
        "number": y -> numberFormat("#").parse($y)+1 });
    $result
}

Ausgabe: [{date= 15.07.24, number= 6.0, name= test}, {date= 16.07.24, number= 3.0, name= test2}, {date= 17.07.24, number=, name=}]

Hier wird beim Aufruf von parseCSV ein Parser für Spalte mit den Namen "date" und ein Parser für Spalte mit den Namen "number" mit angegeben, welche die Einträge aus dem entsprechenden Zellen aufbereitet.
