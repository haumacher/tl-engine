Syntax

	log($message)

log($message, $arguments)

Beschreibung

Schreibt einen Eintrag in das Anwendungslog.

Parameter

Name Typ Beschreibung Pflicht Default
message Zeichenkette Die Message, die ins Log geschrieben werden soll. Falls auch arguments angegeben werden, können in der Message Platzhalte im Form von {0}...{n} definiert werden. Enthält die Message einen Platzhalter, aber es gibt kein Argument, um es zu füllen, wird im Klartext {n} in die Nachricht geschrieben. ja
arguments Menge Argumente, die in der angegebenen Reihenfolge in die Platzhalter von message eingefügt werden. Werden keine oder zu wenige Platzhalter angegeben, dann werden nicht alle Argumente verwendet. nein

Beispiele

Einfache Log Message

	log('Hello');

Ausgabe (im Log): Hello

Platzhalter Log Message

	log('Current date: {0}. Current week day: {1}', today(), today().toUserCalendar().dayOfWeek())

Ausgabe (im Log): Current date: 14.02.22 16:00. Current week day: 2

Platzhalter mit zu wenigen Argumenten

log('Current date: {0}. Current week day: {1}')

Ausgabe (im Log): Current date: {0}. Current week day: {1}

Da keine Argumente angegeben wurden, werden die Platzhalter nicht ersetzt und so ausgegeben.

Platzhalter mit zu vielen Argumenten

log('Current date: {0}', today(), today().toUserCalendar().dayOfWeek())

Ausgabe (im Log): Current date: 14.02.22 16:00

Es gibt mehr Argumente als Platzhalter, daher wird das zweite Argument ignoriert.
