Das Konzept der verteilten Anwendungen kann mit den Modulen com.top_logic.sync.demo.* ausprobiert werden. Um den Anwendungsverbund in der Entwicklungsumgebung zu starten, müssen die folgenden Schritte durchgeführt werden.

Starten von ZooKeeper

In den Debug Configuration in Eclipse findet man im öffnenden Dialog unter Java Application den Eintrag Start ZooKeeper um ​ZooKeeper zu starten.

Starten von Kafka

In den Debug Configuration in Eclipse findet man im öffnenden Dialog unter Java Application den Eintrag Start Kafka um den ​Kafka Server zu starten.

Producer und Consumer

Für eine Demo benötigt man einen Client ​Producer, der Nachrichten verschickt und einen Client ​Consumer, der die Nachrichten empfangen kann. Deswegen müssen zwei weitere Module gestartet werden.

Starten der Kafka Demo für den Producer

In den Debug Configuration in Eclipse findet man im öffnenden Dialog unter Java Application den Eintrag Start Start Top Logic Kafka Demo, um die Kafka Demo für den Producer zu starten. Unter ​http://localhost:8080/kafkaDemo/ findet man nun die Demo für den Producer.

Starten der Kafka Demo für den Consumer

In den Debug Configuration in Eclipse findet man im öffnenden Dialog unter Java Application den Eintrag Start Start Top Logic Kafka Demo (8081), um die Kafka Demo für den Consumer zu starten. Unter ​http://localhost:8081/kafkaDemo/ findet man nun die Demo für den Consumer.
