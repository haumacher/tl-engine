The help editor allows you to create and edit help pages of views. The view is divided into two parts.

On the left side there is a tree that represents the different help pages:

In the toolbar it is possible to switch between the different languages supported for the help. In the menu

there are numerous import and export functions. It is possible to create additional (sub)pages for the selected help page by specifying a title. The action

checks the consistency of the help pages: Check for duplicates, validity of links, etc.

Furthermore, it is possible to mark views during application development so that help pages can be created for them. For example, in the configuration of a tab

there is the possibility to set Enable help. If this configuration option is set, then in the menu of the help editor the action

in the Help Editor menu will generate a help page for the tab. With the command

the selected help page including all subpages would be translated. If you want to translate e.g. German pages into English, you have to switch to the target language first. In the toolbar of the table of contents there is a field to select this language.

Then the pages can be translated by specifying a language from which the pages should be translated automatically.

On the right side a WYSIWYG editor offers the possibility to easily edit the selected help page.

In the menu of the toolbar

you can validate the individual help page separately, i.e. test the validity of the links and have them translated. In addition, it is possible to display the view for which this help page provides the documentation or to compare it with old revisions of the help page using the Compare... action.
