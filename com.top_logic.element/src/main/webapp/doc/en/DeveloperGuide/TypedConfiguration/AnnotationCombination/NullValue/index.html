null als Konfigurationswert

Die Annotation Nullable und NonNullable definieren, ob der Wert null als Propertywert erlaubt ist:

  • Ohne Annotation: Bei Strings, Listen und Maps ist null nicht erlaubt. Bei ConfigItems, ConfiguredInstances usw. ist es erlaubt.
  • Nullable: null ist erlaubt. Kann nicht an primitiven Typen (int, boolean, …), sondern nur an den entsprechenden Objekt-Typen verwendet werden (Integer, Boolean, …).
  • NonNullable: null ist nicht erlaubt.

Der Default-Wert richtet sich dannach ob null erlaubt ist. Ist es erlaubt und kein anderer Default gesetzt, ist null der Defaultwert. Ist null nicht erlaubt, ist der Defaultwert z.B. der leere String, die leere Liste, false bei Booleans, 0 bei Zahlen, ein leeres ConfigItem usw.
