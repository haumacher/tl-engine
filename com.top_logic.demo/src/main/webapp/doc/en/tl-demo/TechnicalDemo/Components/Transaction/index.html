The view shown here demonstrates a user-defined transaction where the user is prompted for an input of additional arguments beforehand.

The button "Duplicate selection" defined here is defined via the configuration option "Add dialog / Transaction with input". The functionality shown as an example duplicates the row object selected in the table and gives it a new name in the process. This new name for the newly created object can be entered by the user in a dialog beforehand. The system proposes a default value for the new name, which is derived from the name of the object to be copied. The name of the original object is prefixed with "Copy of ...":
