An application is able to access an LDAP so that the users existing in the LDAP are mirrored into the application. For this, the corresponding LDAP configurations must be included.

General configuration

The general configuration of the LDAP is defined in Top-Logic in com.top_logic/webapp/WEB-INF/conf/ldapConf.xml. To be able to use it, the name of the file without path must be entered in the metaConf.txt in webapp/WEB-INF/conf of the application that should be able to use the LDAP. 

Specific LDAP configuration

A specific LDAP configuration contains all information about the LDAP which should be connected. This includes the LDAP domain, the LDAP URL and LDAP groups. As a template for the configuration ldapConf.xml can be used. If it is an LDAP of the BOS, the configuration file name must end on .bos.xml.

Configurations  that are not to be delivered with Top-Logic must be stored under deploy/private/webapp/WEB-INF/conf.

The name of the specific configuration must be entered in the metaConf.txt in webapp/WEB-INF/conf  of the application that will use the LDAP.

Note: An LDAP connection is only possible with a license. The demo license is not sufficient, because it allows only one user.

Example

The metaConf.txt of an application can look like this:

...
ldapConf.xml
ldapConf.bos.xml
