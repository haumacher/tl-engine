Die Sicht demonstriert den Zugriff auf das Kontext-Objekt einer Komposition. In dem hier aufgebauten Szenario lassen sich "Welten" erstellen. Jede Welt besteht aus einer Menge von "Dingen". Die Relation Welt zu Dingen ist eine Komposition. Jedes Ding gehört also zu genau einer Welt. Ein Ding kann für andere Dinge benötigt werden. Diese Benötigt-Relation wird über ein separates Fachobjekt dargestellt, da die Relation mit einem Grund annotiert ist. Jedes Link-Objekt der Benötigt-Relation gehört zu genau einem Ding (ebenfalls eine Kompositionsbeziehung).

Bei der Benötigt-Relation muss jetzt sichergestellt werden, dass Dinge einer Welt nur Dinge derselben Welt benötigen dürfen. Daher muss die Auswahl von Dingen in der Benötigt-Relation auf die Dinge derselben Welt eingeschränkt werden, zu der das Ding gehört, für welches ein Benötigt-Link-Objekt angelegt wird. Hierfür muss in dem Option-Provider für das Attribut "Ding" der "Benötigt-Relation" auf den Kontext zugegriffen werden, in dem sich das Link-Objekt befindet. Dies muss sowohl bei der Anlage eines neuen Benötigt-Links als auch bei der Bearbeitung eines bestehenden Links sichergestellt sein.
