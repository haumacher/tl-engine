Das Steuerprogramm dient zur Steuerung von zeitgesteuerten/manuell angestossenen Anwendungen auf einem Server. In den einzelnen Fenster werden die Informationen über die Aufgabe und deren Läufe seit Serverstart dargestellt.

Buttons:

"Aufgabe ausführen" - mit dieser Funktion wird eine Aufgabe manuell, unabhängig von ihrer Einplanung gestartet.

"Aufgabe abbrechen" - mit dieser Funktion kann eine laufende Aufgabe kontrolliert abgebrochen werden.

"Deaktivieren" - mit dieser Funktion wird eine Aufgabe deaktiviert und kann weder eingeplant noch manuell gestartet werden.

"Reaktivieren" - mit dieser Funktion wird eine deaktivierte Aufgabe aktiviert und kann wieder eingeplant oder manuell gestartet werden.

"Clusterlock löschen" - mit dieser Funktion werden Locks des gesamten Clusters gelöscht. In der Regel laufen die Aufgaben auf einem Knoten im Cluster, so dass diese Funktion nicht aktiv ist.

"aktualisieren" - mit dieser Funktion wird die GUI-Anzeige neu aufgebaut und damit die aktuellen Daten angezeigt.

"Fehlgeschlagene Läufe" - mit dieser Funktion erscheint ein Dialog-Fenster in dem alle fehlgeschlagenen Läufe aller Aufgaben seit Serverstart aufgeführt sind.

Ausgewählte Aufgabe: Ergebnisse der letzten Läufe

Dieses Window stellt die bisherigen Läufe der in "Steuerprogramm: Registrierte Aufgaben" selektierte Aufgabe dar, seit Start des Servers.

"Aufgabe" - beschreibt die Tätigkeit des Laufes.

"Cluster Knoten" - Servername auf dem die Anwendung läuft.

"Dauer" - Zeitdauer des Laufes.

"Ende" - Uhrzeit Laufende.

"Ergebnis" - War dieser Lauf der Anwendung "okay"/"notokay".

"Fehler" - Fehler der zum Ergebnis "notokay" führte.

"Logdatei" - Name der Logdatei, die beim Lauf der Anwendung geschrieben wurde.

"Nachricht" - Schlussinformation des Laufes der Aufgabe.

"Start" - Datum/Uhrzeit des Startes des letzten Laufes der Aufgabe.

"Warnungen" - Warnmeldung des Laufes der Aufgabe.

Steuerprogramm

Hier werden die Parameter des Steuerprogrammes dargestellt.

"Thread Name" - beschreibt die Tätigkeit des Laufes.

"Max. Laufzeit eines Jobs" - maximale Laufzeit eines Jobs (Aufgabe), danach wird er mit "timeout" beendet.

"Maximal gleichzeitig" - Anzahl der Jobs (Aufgaben), die gleichzeitg, vom Steuerprogramm überwacht, aktiv sein dürfen.

"Wartend" - Anzahl der Jobs (Aufgaben) in bei "Registierten Aufgaben".

"Aktiv" - Anzahl der Jobs (Aufgaben), die im Augenblick des Aufrufes dieser GUI aktiv waren.

"Letzter Lauf" - Datum des letzten Laufes einer Aufgabe im Steuerprogramm.

geplante Aufgaben

"ausgewählte Aufgabe" - Darstellung der Parameter der in "Steuerprogramm: Registrierte Aufgaben" ausgewählten Aufgabe.

"Ausführungsplan" - Darstellung der Pararmeter der in "Steuerprogramm: Registrierte Aufgaben" ausgewählten Aufgabe, in Bezug auf die Ausführung.

"Status" - Darstellung des Ergebnisses/der Pararmeter der in "Steuerprogramm: Registrierte Aufgaben" ausgewählten Aufgabe, in Bezug auf die letzte/nächste Ausführung.
