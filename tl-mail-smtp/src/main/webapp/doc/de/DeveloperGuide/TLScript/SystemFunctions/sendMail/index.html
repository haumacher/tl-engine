Syntax

sendMail(
  subject: "Hello world!", 
  to: "Mr. Foobar <foo@bar.com>", 
  body: {{{
     <h1>Hello {$receiver}<h1>
     <p>...</p>
  }}},
  attachements: [
    binary(...),
    {
      "cid": "attachement1",
      "data": binary(...)
    }
  ],
  cc: ["info@foo.com", "bar@foo.com"], 
  bcc: "hidden@foobar.com"
)

Beschreibung

Verschickt eine E-Mail mit den angegebenen Parametern. Damit diese Funktion genutzt werden kann, muss der E-Mail-Versand in der Applikation aktiviert sein.

Parameter

Name Typ Beschreibung Pflicht Default
subject Zeichenkette Der Betreff der E-Mail ja
to Zeichenkette/List von Zeichenketten Der oder die Empfänger der E-Mail. Bei einer Massen-E-Mail sollte aus Gründen des Datenschutzes immer nur ein einzelner Empfänger angegeben werden und die Funktion für alle Empfänger einzeln aufgerufen werden. ja
body Zeichenkette/HTML/Binärdaten Der Inhalt der E-Mail. ja
attachements Liste von Binärdaten oder Attachement-Beschreibungen (siehe unten) Zusätzliche Inhalte die in der E-Mail mitversendet werden. nein []
cc Zeichenkette/List von Zeichenketten Liste von Emfängern, die eine Kopie der E-Mail erhalten sollen nein []
bcc Zeichenkette/List von Zeichenketten Liste von Emfängern, die eine Kopie der E-Mail erhalten sollen, ohne dass die anderen Empfänger ihre E-Mail-Adressen zu sehen bekommen. nein []

Rückgabewert

Typ: Wahrheitswert

Ob der E-Mail-Versand erfolgreich war.

Attachement-Beschreibung

Als Attachement kann entweder direkt ein Binärwert übergeben werden, oder ein JSON-Objekt, welches das Attachement näher beschreibt:

{
    "filename": "policies.pdf",
    "disposition": "attachment",
    "description": "Some attachement.",
    "cid": "id1",
    "headers": {
        "key": "value"
    },
    "data": $binaryData
}

Die Attribute der Attachement-Beschreibung haben dabei die folgende Bedeutung:

  • "filename": Der Dateiname für das Attachement, wie er in dem E-Mail-Programm des Empfängers angezeigt werden soll. Beim Speichern des Attachements wird dieser Dateiname dem Nutzer vorgeschlagen.
  • "disposition": "inline" oder "attachement". Bei der Angabe "inline" wird das Attachement vor dem Empfänger versteckt. Das ist sinnvoll, wenn beispielsweise ein Bild mitgeschickt wird, das in den Inhalt der E-Mail eingebettet wird. Siehe "cid".
  • "description": Eine optionale Beschreibung des Attachements.
  • "cid": Die Content-ID des Attachements. Über diese ID kann beispielsweise ein angehängtes Bild in eine HTML-E-Mail eingebettet werden. Wenn als "cid" der Wert "id1" angegeben wurde, kann dieses Attachement beispielsweise über <img src="cid:id1"/> aus dem HTML-Inhalt der E-Mail referenziert werden.
  • "headers": Weitere optionale Header-Informationen, die an dem Attachement gesetzt werden.
  • "data": Der eigentliche Inhalt des Attachements. Der Inhalt kann wie der Inhalt der E-Mail entweder eine Zeichenkette, HTML, oder Binärdaten sein.
