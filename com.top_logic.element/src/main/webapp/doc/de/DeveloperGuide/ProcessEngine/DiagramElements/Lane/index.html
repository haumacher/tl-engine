Eine Bahn einer Kollaboration repräsentiert eine Rolle an die eine Aufgabe zugewiesen wird, bzw. die eine Aufgabe bearbeiten darf. Die Eigenschaften der Bahn sind über einen Klick auf den Hintergrund der Bahn zu erreichen.

Die Bahn ist die zentrale Steuerung von Berechtigungen in einer prozessgetriebenen Applikation. Die Abbildung der Prozessrolle "Actor 1" in obigem Beispiel auf die Nutzergruppe "Users" im System wird über die Eigenschaften der Bahn vorgenommen. Es gibt prinzipiell zwei Möglichkeiten, um diese Abbildung von Prozessrolle auf Systemnutzer vorzunehmen.

Statische Rollenvergabe

Durch Zuweisung einer Nutzergruppe zu einer Bahn kann eine statische Zuweisung von Prozessrolle zu Nutzern im System vorgenommen werden. In diesem Fall dürfen alle Nutzer in der angegebenen Nutzergruppe eine Aufgabe in dieser Bahn übernehmen, bzw. sind für ihre Durchführung verantwortlich. Diese Rollenvergabe ist statisch, weil sie alle Aufgaben in einer Bahn an dieselben Nutzer zuweist unabhängig von den Daten des Prozessdatenobjektes.

Eine solche statische Rollenvergabe ist insbesondere für den Start-Event einer Kollaboration sinnvoll. Die Bahn, in der der Start-Event liegt, steuert, welche Nutzer in ihrem Prozess-Cockpit einen neuen Prozess starten können. Da zu diesem Zeitpunkt noch keine Daten vorliegen, kann diese Berechtigung nur über eine Nutzergruppe berechtigt werden.

Dynamische Rollenvergabe

Die Vergabe einer Aufgabe kann von den Daten im Prozeddatenobjekt abhängig gemacht werden. Wird beispielsweise ein Urlaubsantrag eingereicht, so könnte der Abteilungsleiter der Abteilung des Nutzers, der den Antrag stellt, für die Genehmigung zuständig sein. Eine so dynamische Zuweisung von Augaben an Systemnutzer kann über eine Aktor-Funktion erreicht werden. Diese Funktion erhält das Prozessdatenobjekt und berechnet daraus den oder die Nutzer, denen die Aufgabe zugewiesen werden soll.
