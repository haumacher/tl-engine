Die Sicht stellt die Demodaten in einer einfachen Tabelle dar.

Die Konfiguration einer Tabelle gliedert sich größtenteils in drei Bereiche.

kann das Komponentenmodell der Tabelle konfiguriert werden. Bspw. kann die Tabelle über einen Kanal mit weiteren Komponenten verbunden werden.

Zusätzlich kann man die unterstützten Typen der Tabelle und die Spalten, die standardmäßig in der Tabelle dargestellt werden sollen, konfiguriert werden. Die ID-Spalte gibt an nach welcher Spalte initial sortiert wird.

Durch die folgende Konfiguration wird das Modell der Tabelle festgelegt.

Hier wird , ähnlich wie bei Baum, das Komponentenmodell ignoriert. In der Tabelle werden alle Elemente des Typs DemoTypes.All angezeigt. Das Zusammenspiel von der Komponentenmodell für Listenelement und Prüfer für Benutzung als Komponentenmodell Konfiguration ist insbesondere von Bedeutung, um zu entscheiden, falls in einer anderen Sicht ein Objekt erzeugt wurde, ob dieses Objekt auch in dieser Tabelle automatisch hinzugefügt bzw. dargestellt wird.
