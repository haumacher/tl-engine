Zweck

  • Der Gui Inspector  dient der Inspektion der GUI ohne den HTML-Code der Seite analysieren zu müssen.
  • Außerdem wird er verwendet um Assertions für Automatische Anwendungstests aufzuzeichnen.

Verwendung

  • Gui Inspector  in der persönlichen Konfiguration aktivieren:
  • <section name="Debugging" comment="Contains switches for debugging">
        <entry name="inspectEnabled" value="true"
                comment="Enables the gui inspection features. They are also used to record assertions for automated application tests. Therefore, they are also enabled if the ScriptingRecorder is enabled. See section 'ScriptingRecorder', entry 'enabled'." />
</section>
  • Der Gui Inspector  wird außerdem aktiviert, wenn der ScriptingRecorder aktiviert wird.
  • Anwendung starten
  • Der Gui Inspector  wird mittels eines Buttons in der Main-Tabbar oder der Titelzeile eines Dialogs aufgerufen
  • Der Mauszeiger wird zu einem Zeiger mit Fragezeichen - Hilfecursor (mehr passiert nicht)
  • Auf das zu untersuchende Element klicken
  • Der Gui Inspector öffnet sich und zeigt Informationen über das zu untersuchende Element an.
    • Falls nicht, wurde das Element nicht erkannt. Einen Monat Warten und erneut versuchen.
  • Beim Aufzeichnen von Anwendungstests:
    • Die gewünschten Assertions auswählen und auf "Assertions aufzeichnen" klicken
    • Der Gui Inspector verschwindet wieder

Tipps

  • Wenn der Gui Inspector Probleme hat ein Field direkt zu inspizieren, kann es helfen statt dessen das Label des Field zu inspizieren. Dadurch wird das Field selber inspiziert.
  • Bei Tabellen die ein Label haben kann man zum Teil das Label inspizieren und dadurch das Field inspizieren, das durch die Tabelle repräsentiert wird.

  • Field
  • Labels von Fields (Es werden die Informationen zum Field angezeigt)
  • Werte in Tabellen (Auch Grid  & FrozenTable)
