Für eine Nutzersitzung gibt es eine maximale Zeitspanne, welche die Sitzung aktiv bleibt, ohne dass eine Nutzerinteraktion stattgefunden hat. Der Sitzungsablauf wird normalerweise vom Anwendungscontainer gesteuert, kann aber in der web.xml der Anwendung überschrieben werden. Hierfür muss in der Datei /webapp/WEB-INF/web.xml im Modul der eigenen Anwendung der folgende Bereich aktiviert werden:

<session-config>
   <session-timeout>20</session-timeout>
</session-config>

Das Element session-timeout definiert das Standard-Timeout-Intervall für alle Sitzungen, die in dieser Webanwendung erstellt werden. Der angegebene Timeout muss als ganze Zahl von Minuten ausgedrückt werden. Wenn der Timeout 0 oder weniger ist, stellt der Container sicher, dass Sitzungen nie ablaufen. Wenn dieses Element nicht angegeben wird, legt der Container eine Standard-Timeout-Periode fest.
